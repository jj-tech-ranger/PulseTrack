# PulseTrack - Fitness Tracking Mobile App

![PulseTrack Logo](assets/images/logo.svg)

## Overview

PulseTrack is a comprehensive fitness tracking mobile application built with Flutter (Dart) and Oracle Database. The app features workout tracking, progress monitoring, personalized fitness plans, and social features to help users achieve their fitness goals.

## Features

- **User Authentication**: Secure login and signup with email/password
- **Profile Management**: Customizable user profiles with fitness goals
- **Workout Tracking**: Log workouts, exercises, sets, and reps
- **Progress Monitoring**: Track weight, measurements, and fitness milestones
- **Exercise Library**: Comprehensive database of exercises with instructions
- **Custom Workouts**: Create and save personalized workout routines
- **Statistics & Analytics**: Visual charts and graphs of fitness progress
- **Dark Theme**: Modern navy blue and pastel blue color scheme

## Technology Stack

- **State Management**: Provider

## Setup Instructions

### Prerequisites

- Flutter SDK (3.0.0 or higher)
- Dart SDK (2.17.0 or higher)
- Oracle Database 19c
- Node.js 16.x or higher (for backend)
- Android Studio / Xcode (for mobile development)

### Installation Steps

1. **Clone the repository**
```bash
git clone https://github.com/jj-tech-ranger/PulseTrack.git
cd PulseTrack
```

2. **Install Flutter dependencies**
```bash
flutter pub get
```

3. **Configure API endpoints**
Edit `lib/config/api_config.dart` to point to your backend server:
```dart
static const String baseUrl = 'http://your-server-ip:3000/api';
```

4. **Run the app**
```bash
flutter run
```

## Oracle Database Schema

### Tables

#### 1. USERS Table
```sql
CREATE TABLE users (
    user_id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    email VARCHAR2(255) UNIQUE NOT NULL,
    password_hash VARCHAR2(255) NOT NULL,
    full_name VARCHAR2(255) NOT NULL,
    date_of_birth DATE,
    gender VARCHAR2(20),
    height NUMBER(5,2),
    current_weight NUMBER(5,2),
    goal_weight NUMBER(5,2),
    fitness_level VARCHAR2(50),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_users_email ON users(email);
```

#### 2. WORKOUTS Table
```sql
CREATE TABLE workouts (
    workout_id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id NUMBER NOT NULL,
    workout_name VARCHAR2(255) NOT NULL,
    workout_date DATE NOT NULL,
    duration_minutes NUMBER,
    calories_burned NUMBER,
    notes CLOB,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(user_id) ON DELETE CASCADE
);

CREATE INDEX idx_workouts_user ON workouts(user_id);
CREATE INDEX idx_workouts_date ON workouts(workout_date);
```

#### 3. EXERCISES Table
```sql
CREATE TABLE exercises (
    exercise_id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    exercise_name VARCHAR2(255) NOT NULL,
    category VARCHAR2(100),
    muscle_group VARCHAR2(100),
    equipment VARCHAR2(100),
    difficulty VARCHAR2(50),
    instructions CLOB,
    image_url VARCHAR2(500),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_exercises_category ON exercises(category);
CREATE INDEX idx_exercises_muscle ON exercises(muscle_group);
```

#### 4. WORKOUT_EXERCISES Table
```sql
CREATE TABLE workout_exercises (
    workout_exercise_id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    workout_id NUMBER NOT NULL,
    exercise_id NUMBER NOT NULL,
    sets_count NUMBER,
    reps_count NUMBER,
    weight_kg NUMBER(5,2),
    rest_seconds NUMBER,
    notes VARCHAR2(500),
    FOREIGN KEY (workout_id) REFERENCES workouts(workout_id) ON DELETE CASCADE,
    FOREIGN KEY (exercise_id) REFERENCES exercises(exercise_id)
);

CREATE INDEX idx_workout_ex_workout ON workout_exercises(workout_id);
```

#### 5. PROGRESS_TRACKING Table
```sql
CREATE TABLE progress_tracking (
    progress_id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id NUMBER NOT NULL,
    tracking_date DATE NOT NULL,
    weight_kg NUMBER(5,2),
    body_fat_percent NUMBER(4,2),
    chest_cm NUMBER(5,2),
    waist_cm NUMBER(5,2),
    hips_cm NUMBER(5,2),
    biceps_cm NUMBER(5,2),
    thighs_cm NUMBER(5,2),
    notes VARCHAR2(500),
    FOREIGN KEY (user_id) REFERENCES users(user_id) ON DELETE CASCADE
);

CREATE INDEX idx_progress_user ON progress_tracking(user_id);
CREATE INDEX idx_progress_date ON progress_tracking(tracking_date);
```

#### 6. USER_GOALS Table
```sql
CREATE TABLE user_goals (
    goal_id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id NUMBER NOT NULL,
    goal_type VARCHAR2(100) NOT NULL,
    target_value NUMBER(10,2),
    current_value NUMBER(10,2),
    target_date DATE,
    status VARCHAR2(50) DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(user_id) ON DELETE CASCADE
);
```

### Database Initialization Script

Create a file `database/init_db.sql` with all the above table definitions and run:

```bash
sqlplus username/password@database < database/init_db.sql
```

## Backend API Implementation

### Required API Endpoints

#### Authentication
- `POST /api/auth/signup` - Create new user account
- `POST /api/auth/login` - Authenticate user
- `POST /api/auth/logout` - End user session
- `GET /api/auth/profile` - Get user profile

#### Workouts
- `GET /api/workouts` - List user workouts
- `POST /api/workouts` - Create new workout
- `GET /api/workouts/:id` - Get workout details
- `PUT /api/workouts/:id` - Update workout
- `DELETE /api/workouts/:id` - Delete workout

#### Exercises
- `GET /api/exercises` - List all exercises
- `GET /api/exercises/:id` - Get exercise details
- `GET /api/exercises/category/:category` - Filter by category

#### Progress
- `GET /api/progress` - Get user progress history
- `POST /api/progress` - Log new progress entry
- `GET /api/progress/stats` - Get progress statistics

### Sample Backend Setup (Node.js + Express)

```javascript
// server.js
const express = require('express');
const oracledb = require('oracledb');
const app = express();

app.use(express.json());

// Oracle connection pool
oracledb.createPool({
  user: 'your_user',
  password: 'your_password',
  connectString: 'localhost:1521/ORCLPDB1'
});

app.post('/api/auth/signup', async (req, res) => {
  const { email, password, name } = req.body;
  // Implementation here
});

app.listen(3000, () => {
  console.log('Server running on port 3000');
});
```
